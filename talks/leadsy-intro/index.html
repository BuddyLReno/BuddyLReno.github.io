Leadsy

A jQuery plugin for submitting leads to the lead service.

Not to be confused with


Dave "Leedsy" Leeds

Where did this idea come from?

A file named lead.js was used a few times:

  • Smart Money Smart Kids
  • Business Gets Personal
  • EntreLeadership Summit
  • ChrisHogan360.com

How it worked

A singleton that re-initializes itself on button click.

Submission data was hardcoded.

Only one instance of a form could be on a site.

The problem

Chris Hogan's site needed two forms submitting different data for different purposes.

The Solution

Leadsy

  • Initialize once for all forms.
  • Pull data from each form or pass in to an initializer.
  • Allow multiple forms to be used independently.

One Init To Rule Them All

Grab multiple forms and initialize them at the same time.


$("#form1, #form2").leadsy({
  leadServiceEnvironment: "test",
  interestedIn: "urn:leadsy:example"
});

Each instance of Leadsy is stored as data on each form.


var instance1 = $("#form1").data("leadsy");
var instance2 = $("#form2").data("leadsy");

//You can call Leadsy specific functions for each instance.
var form1Data = instance1.scrape();
var form2Data = instance2.scrape();

Combine data specific to each form

Set data multiple ways:

  • Init
  • Data Attributes
  • Combination

Init


$("#form1").leadsy({
  leadServiceEnvironment: "production",
  interestedIn: "urn:lampo:christy-wright:monthly-newsletter"
});

Data Attributes


<form id="form1"
  data-lead-service-environment="production"
  data-interested-in="urn:lampo:christy-wright:monthly-newsletter">

Both


$("#form1").leadsy({
  leadServiceEnvironment: "production"
});
						

<form id="form1"
  data-interested-in="urn:lampo:christy-wright:monthly-newsletter">

Data Attributes Trump Script Level Settings


$("#form1, #form2").leadsy({
  leadServiceEnvironment: "production",
  interestedIn: "urn:leadsy:script-example"
});
						

<form id="form2"
  data-interested-in="urn:leadsy:data-attribute-example">

#form1 will use "urn:leadsy:script-example"

#form2 will use "urn:leadsy:data-attribute-example"

Separate Forms, Separate Data

Each form will submit it's own data, allowing each form to work independently.


<form id="form1">
  <input type="text" id="email" data-leadsy-field="email">
  <input type="text" id="lastName" data-leadsy-field="lastName">
</form>
						

{
  contact: {
    email: {
      primary: "bwayne@daveramsey.com"
    },
    last_name: "Wayne"
  }
}
						

<form id="form2">
  <input type="text" id="email" data-leadsy-field="email">
</form>
						

{
  contact: {
    email: {
      primary: "bwayne@daveramsey.com"
    }
  }
}

Complicated Simplicity

Can't remember how to structure your data
for the lead service?

Leadsy takes your flattened form data and creates the complicated structure that the lead service requires.


var postedDate = {
  contact: {
    email: {
      primary: "hal.jordan@daveramsey.com"
    },
    first_name: "Hal",
    last_name: "Jordan",
    address: {
      primary: {
        postal_code: "37027"
      }
    }
  },
  interested_in: "urn:leadsy:example-data",
  submission_data: {
    newsletter_lists: "leadsy-example-list"
  }
};

No Fields Left Behind

Any field that is not a named lead service field will be added to the lead's "submission_data" bucket.


var formData = {
  contact: {
    email: {
      primary: "donald.blake@daveramsey.com"
    },
    first_name: "Donald",
    last_name: "Blake"
  },
  submission_data: {
    gender: "male"
  }
};

Subscriptions & Triggered Sends


$("#form1").leadsy({
  newsletterLists: "list-example",
  gateway: "exact_target",
  templateKey: "exact_target_trigger"
});
						

<form id="form2"
  data-newsletter-lists="different-list,another-list"
  data-gateway="exact_target",
  data-template-key="a_different_ET_trigger">
